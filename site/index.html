Volunteer scheduling that schedules itself.

SignUpFlow auto-generates fair rosters for churches, sports leagues, and non-profits — a constraint-based solver does the hard part while volunteers self-serve open shifts and cover each other's swaps.

View on GitHub See what it does

Everything a roster needs, end to end

From the first invite to a published, calendar-synced schedule — with the scheduling itself automated.

⚙︎

Auto-scheduling solver

A greedy heuristic solver with constraint-based optimization builds fair schedules across roles and people automatically.

◷

Availability & time-off

Volunteers set time-off and recurring (RRULE) unavailability; the solver respects every block.

↻

Self-serve & swaps

Volunteers claim open shifts and cover each other's swap requests without an admin in the loop.

⛁

Teams, roles & tenancy

Role-based access (admin / volunteer) and strict per-org isolation so every organization stays separate.

∑

Constraints DSL

Express scheduling rules — spacing, eligibility, caps — in a compact constraint language the solver evaluates.

⧉

Calendar sync & analytics

Each volunteer gets an ICS subscription of published shifts, plus volunteer and event analytics.

{ }

API-first & CLI

Headless by design: a full REST API plus a YAML-in / JSON-out CLI for scripted, reproducible scheduling.

✉

Invitations & inbox

Token-based invites onboard people in one tap; an in-app inbox keeps assignees notified.
